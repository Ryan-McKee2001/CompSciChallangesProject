How to guide- Getting started with enable3d

Hi, and welcome to my guide on getting started with enable3d(https://enable3d.io/). Enable3d is an open-source and free library created to allow users to easily add physics through the use of physics to graphics in a web browser. We do this through the use of two libraries. The physics library called ammo.js is a javascript port of the popular c physics engine bullet physics and three.js a popular javascript graphics library that will be used in conjunction with ammo.js (https://github.com/kripken/ammo.js/) to create a representation of a physics world in a browser. three.js is a very popular graphics library for javascript used by many great organizations like NASA, tesla, space x, Maserati this library has many online examples and tutorials which you can find on this website: https://threejs.org/.

Rigid bodies

Dynamic Rigid Body

What is a Dynamic Rigid Body?

During physics interaction, there has to be a body. In ammo.js this body is called a rigid body/ collision body. A rigid body is a thing that moves, collides, has mass, and can have impulses applied to it. But on its own, it's like a ghost without a shell. It needs a shape so as to interact in collisions and to also help calculating its inertia tensor( distribution of mass )

How to create a Dynamic rigid body?

To add a Dynamic rigid body using enable3d you just enter this statement into the create function:

let box1 = this.physics.add.box({}, { phong: { color: 'green' } })

This is what should show up in your scene:

this statement can be edited to create multiple different types of shapes, colours and materials.

let sphere= this.physics.add.sphere({}, { lambert: { color: 'red' } })

For a full list of colours, shapes, and materials you can use for the objects you add to the world you can go to three.js documentation.

Kinematic Rigid Body

What is a Kinematic rigid body?

These bodies are not affected by the physics of the Ammo.js world but can be animated during the physics simulation, These bodies are similar to the static rigid bodies in that they have a mass of 0 and are not affected by the physics of the world, but unlike the static rigid bodies we animate these bodies.

How to create a Kinematic rigid body?

To create a Kinematic body we just create a body in the create function with a mass of 0 like how we would create a static body. Pass it into the update function and update its properties.

Static Rigid body

What is a Static Rigid Body?

These bodies are not affected by the physics of the Ammo.js world they have a mass of 0.

How to create a static rigid body?

To create a static body it is similar to creating any other body you just need to give the static objects mass property to 0 in the parameters when creating the object.

let staticbox = this.physics.add.box({y:4, mass:0}, {phong: { color: 'yellow' }})

// to show that the box is static I just a dynamic rigid body to drop on top of it:

let box = this.physics.add.box({y:6})

Constraints

What are constraints?

Constraints can be thought of as joints that would be used in real life to connect multiple objects. There are multiple types of constraints in ammo.js to carry out different functions.

Point to Point constraint

What is a Point to Point constraint?

Point-to-point constraints limit the translation of two pivot points of rigid bodies to match the world space. Using this constraint, you can create a chain of rigid bodies. This constraint can be useful for creating something like a robotic arm using several rigid bodies in columns and rows.

How to create a Point to Point constraint?

To create a point-to-point constraint you need to start by creating 2 or more bodies with some distance between them in this case I have chosen to change the z-axis for box 2 to 4.

let box1 = this.physics.add.box({ y: 2, mass: 0 }) // the mass property being equal // 0 means the body is static

let box2 = this.physics.add.box({ y: 2, z: 4 })

Now you can create the point-to-point constraint to do that we call the constraints.PointToPoint function passing into the parameters both the boxes and the pivot points for each.

this.physics.add.constraints.pointToPoint(box1.body, box2.body, {

pivotA: { z: 1.5 },

pivotB: { x: -1.5 }})

Now your scene should look like this when the debugger is on:

You can create more than one connection with a point to point constraint to create a chain

// creating the bodies

let box1 = this.physics.add.box({ y: 6, mass: 0 })

let box2 = this.physics.add.box({ y: 6, z: 4 })

let box3 = this.physics.add.box({y:6, z: 6})

// Creating p2p between box 1 and 2

this.physics.add.constraints.pointToPoint(box1.body, box2.body, {

pivotA: { z: 1.5 },

pivotB: { x: -1.5 }})

// Creating p2p between box 2 and 3

this.physics.add.constraints.pointToPoint(box2.body, box3.body, {

pivotA: { z: 1.5 },

pivotB: { x: -1.5 }})

Hinge Constraint

What is a hinge constraint?

Hinge constraints are joints that restrict rotation around the pivot of two bodies to only one axis, therefore this axis could be useful for creating something like a door. Or even flaps on an airplane model as this constraint also allows the user to specify the limits and motor of the hinge.

How to add a Hinge constraint?

Adding a hinge constraint similar to any other constraint being added although the main difference being 2 of the parameters it takes. axisA and axisB

// Creating the bodies

let box1 = this.physics.add.box({ depth: 0.25, y: 5, z: 0, x: 0, mass: 0 }) //static body

let box2 = this.physics.add.box({ depth: 0.25, y: 5, z: 0.5, x: 0 + 1.25 })

let box3 = this.physics.add.box({ depth: 0.25, y: 5, z: 1, x: 0 + 1.25 })

// Creating the first constraint between box1 and 2

this.physics.add.constraints.hinge(box1.body, box2.body, {

pivotA: { y: -0.65 },

pivotB: { y: 0.65 },

axisA: { x: 1 },

axisB: { x: 1 }

})

// creating the second constraint between box 2 and 3

this.physics.add.constraints.hinge(box2.body, box3.body, {

pivotA: { y: -0.65 },

pivotB: { y: 0.65 },

axisA: { x: 1 },

axisB: { x: 1 }

})

the scene should now look like this when the objects with the hinge constraints are added:

Slider constraint

What is a slider constraint?

The slider constraint allows a body to rotate around one axis and translate along that axis. This constraint would be useful for creating something like a piston.

How to add a slider constraint?

Creating a slider constraint is very similar to the creation of any other constraint. The main difference is that 2 of the parameters that the slider constraint takes when creating it is a lower and upper limit which is how far up or down along the reference frame that the second body can slide.

// Create main cylinder

let box1 = this.physics.add.cylinder({ height: 6, y: 10, x: 0, radiusTop: 0.6, radiusBottom: 0.6, collisionFlags: 6, mass: 100 })

// create second cylinder

let box2 = this.physics.add.cylinder({ height: 4, y:10, x: 0, radiusTop: 0.4, radiusBottom: 0.4, collisionFlags: 4, mass: 1 })

// takes four parameters the 2 bodies and the frames of those bodies

this.physics.add.constraints.slider(box1.body, box2.body, {

frameA: { x: Math.PI / 2, y: 0, z: 0 },

frameB: { x: Math.PI / 2, y: 0, z: 0 },

linearLowerLimit: -5,

linearUpperLimit: 5

})

Cone twist constraint

What is a cone twist constraint?

This cone constraint is a special version of the point-to-point constraint that adds cone and twist axis limits. The axis for this constraint serves as a twist axis. This constraint is useful for creating something like a ragdoll.

How to add a cone twist constraint?

This is the code for creating a cone twist constraint the main difference is that in the parameters is takes the frames of both the bodies.

// Creating the boxes

let box1 = this.physics.add.box({ y: 5, x: 0, collisionFlags: 2, mass: 100 })

let box2 = this.physics.add.box({ y: 4, x: 0, z: 0 })

// Adding the constraint

this.physics.add.constraints.coneTwist(box1.body, box2.body, {

frameA: { x: 1, y: 1, z: 1 },

frameB: { x: 0.5, y: -0.5, z: 0.5 }

})

Spring constraint

What is a Spring constraint?

A spring constraint applies forces to the attached bodies based on spring and damper behavior this kind of constraint has the behavior of a real-life spring and has many properties that can allow the spring behavior to change a great deal. This is useful for the creation of stuff like car simulations that require spring behavior in their suspensions.

How to create a Spring constraint?

To create a spring constraint you need to define bodies which are going to be attachments for the springs, define the limits on the vector3 which the spring can apply its elastic behaviour and set a damping for the spring these will be all taken as parameters in the creation of the constraint.

// Creating the bodies

let box1 = this.physics.add.box({ y: 10, x: 5, z: 0, mass: 0 })

let box2 = this.physics.add.box({ y: 9, x: 5, z: 0, mass: 2 })

let box3 = this.physics.add.box({ y: 8, x: 5, z: 0, mass: 8 })

// Defining the limits

const linearLowerLimit = { x: -100, y: -100, z: -100 }

const linearUpperLimit = { x: 100, y: 100, z: 100 }

// Adding constraint between boxA and B

this.physics.add.constraints.spring(box1.body, box2.body, {

damping: 250,

linearLowerLimit,

linearUpperLimit

})

// Adding constraint between boxB and c

this.physics.add.constraints.spring(box2.body, box3.body, {

damping: 20,

linearLowerLimit,

linearUpperLimit

})

Compound objects

What is a compound object

compound objects are 2 or more objects that are bound together this can be used to create complex objects that could not be created with the standard built-in bodies.

How to create a compound object

the first step to creating a compound shape is creating a group and setting the reference point for the group.

let group = new THREE.Group()

group.position.z = 9

group.position.y = 3

group.rotation.z -= 1.5

Now we need to create the body's that we are going to add to the group to do this we just declare rigid bodies and set the vector3 in consideration of the reference frame we set for the group.

let c1 = this.add.box({ x: -1, y: -1 })

let c2 = this.add.box({ x: -1, y: 0 })

let c3 = this.add.box({ x: 1, y: 0 })

Now we add the objects we want to group the the group itself

this.add.existing(group)

group.add(c1 as any)

group.add(c2 as any)

group.add(c3 as any)

Finally, If you want to add physics to this group you can add the following statement:

this.physics.add.existing(group as any)

Object properties

Bounciness

setting the bounciness is one of the functions available using three.js to set the bounciness of an object you just need to use this statement:

let sphere = this.physics.add.sphere({x:0, y:10, z:0});

sphere.body.setBounciness(1); // the parameter takes the bounciness

If you enter bounciness to be equal to 1 the bounce will be infinite at the same height it was dropped, If the bounce is higher than 1 it will continuously bounce higher, and if it is less it will bounce lower and lower.

Force

Another property you can change is the force applied to the body. Force can be applied to all 3 vector 3 axis if can be done using the following statements:

sphere1.body.applyForceX(0.3)

sphere.body.applyForceY(-5)

sphere.body.applyForceZ(7)

the force applied to be thought of like wind blowing the object the greater the wind speed the further and faster an object will be blown. The same applies here the greater the force applied to an axis the faster and further the body will be pushed in that direction.

Physics Debugger

A cool feature included within enable3d is a built-in physics debugger. This debugger can allow you to see the axis and the masks of the objects within the physics world which can assist in the detection and correction of errors in your computer program.

Adding a debugger

Adding the debugger is a simple one-line statement within the async create function just add this statement this will enable the physics debugger within the world..

// enable physics debug

if (this.physics.debug) {

this.physics.debug.enable()

}
